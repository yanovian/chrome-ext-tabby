# Tabby

**A cat lives in your browser.**

Tabby floats on the pages you visit, reacts to what you read, and remembers your
online life together — privately, on your device. No cloud. No accounts. No guilt.

## Quick start (development)

```bash
make dev
# or: pnpm dev
```

Keep that command running. WXT watches your files and hot-reloads the extension:

- **Content script / CSS / popup changes** → matching tabs reload automatically
- **Background changes** → the extension reloads itself
- **Manual reload** → `Alt+R` in the dev browser (or press `o` + Enter in the terminal to reopen it)

On first run, WXT opens a Chrome window with Tabby already loaded from `.output/chrome-mv3-dev`.
Prefer that window for day-to-day dev — hot reload is most reliable there.

If you load **`.output/chrome-mv3-dev`** manually in your `chrome://extensions` instead:

1. Keep `make dev` running in a terminal (required for hot reload).
2. Load **`.output/chrome-mv3-dev`** only — not `chrome-mv3` (production).
3. After a save, wait for the terminal to show `Reloaded:` — then refresh the page if Tabby did not return on her own.

Tabby re-registers her content script on every dev service-worker restart, so she should stay installed even when the extension reloads itself.

### Dev mode tips

When running `pnpm dev`, dev interactions are enabled by default:

- Faster stat changes and shorter appearance cooldowns
- **Force tick & speech** button in the settings popup
- Tune dev frequency in settings (max appearances, cooldown, stat multiplier)

To test moods quickly:

1. Open settings → enable dev mode options
2. Browse a docs site (nourishing) or a social feed URL (draining) for a minute
3. Click **Force tick & speech** or switch tabs to flush observations

## Scripts

| Command | Description |
|---------|-------------|
| `make dev` | Dev server with hot reload — leave it running while you edit |
| `make build` | Production build |
| `make assets` | Download bundled ONNX runtime + local speech model (offline) |
| `make check` | Typecheck + lint + test + build |
| `make sprites` | Strip sprite backgrounds / optimize PNGs |
| `make help` | Show all Makefile targets |

Or use pnpm directly: `pnpm dev`, `pnpm check`, etc.

### Life stages

Tabby grows through three visual ages, each with **its own mood sprites**:

| Stage | When | Look |
|-------|------|------|
| **Newborn kitten** | First 14 days | Tiny, fragile, just born |
| **Playful kitten** | Days 15–120 | Small, naughty, energetic |
| **Grown-up Tabby** | Day 121+ | Full adult cat |

In dev mode, use **Dev life stage override** in settings to preview any age instantly.

### Local speech (offline AI)

Tabby’s dialogue is generated by a **small bundled text model** (`flan-t5-small` via
Transformers.js + ONNX), running in an offscreen document. **No network calls at
runtime** — models are downloaded at build/install time into `public/models/` and
`public/ort/` (same pattern as the Breadcrumb extension).

- Toggle **Varied local speech** in settings to switch between AI lines and curated fallbacks
- First load may take a few seconds while the model warms up
- If generation fails or times out, Tabby falls back to hand-written lines

### Using Tabby

- **Drag** Tabby by the `⋮⋮` handle to move her anywhere on the page
- **Tap Tabby** to open mood-aware actions:
  - Stressed → “Why are you frustrated?”
  - Hungry → “Feed Tabby”
  - Bored → “Play with Tabby”
  - Always → Pet, Hide for now
